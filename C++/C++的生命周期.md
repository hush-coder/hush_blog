# C++ 对象生命周期机制

对象的生命周期从底层看是内存分配、构造、析构、释放四个阶段与硬件、运行时环境交互的过程。下面从存储区域、对象创建与销毁的汇编/ABI细节、编译器生成代码的行为等方面展开。

# 1. 对象的存储区域与内存布局

C++ 对象的存储位置决定其生命周期的起始与结束：

| 存储区域 | 生命周期特征 | 内存分配时机 | 释放时机 |
| :--- | :--- | :--- | :--- |
| **栈（自动存储）** | 由作用域控制，编译器自动管理 | 进入作用域时，栈指针（SP）向下移动分配空间 | 离开作用域时，栈指针恢复，析构函数被调用 |
| **静态存储** | 程序启动时分配，结束前释放 | 编译期确定，在数据段（`.data/.bss`）或动态加载时分配 | 程序退出时，由运行时库调用析构函数 |
| **堆（动态存储）** | 程序员手动控制，或由智能指针管理 | 通过 `malloc/new` 从堆管理器申请 | 通过 `free/delete` 归还，或由 RAII 自动触发 |
| **线程局部存储（TLS）** | 线程生命周期内有效 | 线程创建时分配 | 线程退出时释放 |

## 1.1 栈内存布局与构造/析构时机

典型函数调用栈帧（`x86-64`，`GCC/Clang` 的 `Itanium ABI`）：

```text
高地址
+-----------------+
| 参数区域        |
+-----------------+
| 返回地址        |
+-----------------+
| 上一帧基指针    | ← RBP (帧基指针)
+-----------------+
| 局部对象1       | ← 构造在此位置
| 局部对象2       |
| ...             |
+-----------------+ ← RSP (栈指针)
低地址
```

编译器在进入函数时，会依次计算偏移，调用每个局部对象的构造函数（通过 call 指令）。离开作用域时，按构造的逆序调用析构函数。这些调用是显式嵌入在生成的汇编代码中的。

**示例**：函数中有两个局部对象 A a; B b;，编译器生成的伪代码：

```text
push rbp
mov rbp, rsp
sub rsp, 16          ; 分配栈空间
lea rdi, [rbp-8]     ; 获取 a 的地址
call A::A()          ; 构造 a
lea rdi, [rbp-16]
call B::B()          ; 构造 b
...
; 离开作用域时
lea rdi, [rbp-16]
call B::~B()         ; 析构 b
lea rdi, [rbp-8]
call A::~A()         ; 析构 a
mov rsp, rbp
pop rbp
ret
```

## 1.2 静态存储对象的构造与析构
- **全局对象**：构造函数在 `main` 之前（由 `C++` 运行时库 `__cxx_global_var_init` 调用链执行），析构函数在 main 之后（通过 atexit 注册的析构函数）。
- **函数内静态局部对象**：第一次执行到其定义时构造，线程安全（C++11 保证）；析构在程序退出时发生（同样通过 atexit）。

底层实现：编译器为每个静态局部对象生成一个守卫变量（guard），通过原子操作确保单次初始化。例如：

```cpp
void foo() {
    static X x;
}
```

GCC 生成的汇编会调用 `__cxa_guard_acquire` / `__cxa_guard_release` 来实现线程安全的延迟初始化。

# 2. 动态内存分配的底层实现

## 2.1 new 与 delete 的分解

***`new T(args)` 底层执行：***

1. **分配内存**：调用 `operator new`（通常封装 `malloc`）或重载版本，返回 `void*`。
2. **构造对象**：在内存上调用构造函数（通过 `placement new` 或编译器直接调用）。
3. 返回指针。

***`delete p` 执行：***

1. **析构对象**：调用析构函数。
2. **释放内存**：调用 `operator delete`（通常封装 free）。

```cpp
// 编译器将 new T(args) 转换为类似：
void* mem = operator new(sizeof(T));
T* ptr = new(mem) T(args);
return ptr;

// delete ptr 转换为：
ptr->~T();
operator delete(ptr);
```

## 2.2 operator new 与内存分配器

标准库的 operator new 最终调用 malloc（或平台特定分配器）。malloc 从堆中申请内存，返回的内存块通常带有头部元数据（如块大小、标志）。分配器维护空闲链表或采用更复杂的算法（如 dlmalloc、ptmalloc2）。动态分配内存的释放必须与分配大小匹配，这是 C++ 运行时保证的。
