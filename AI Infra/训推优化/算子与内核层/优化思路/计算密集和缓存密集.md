- **计算密集型：**算子的执行时间主要花费在计算上。性能瓶颈是 算力，即处理器的计算核心（FLOPS，每秒浮点运算次数）。常见的例子：大尺寸的矩阵乘法（GEMM）、卷积（Conv）、各种非线性激活函数。
- **访存密集型：**算子的执行时间主要花费在数据读取和写入上。性能瓶颈是 带宽，即内存/缓存的数据传输速率。常见的例子：逐元素操作（加法、乘法）、Reduce操作（求和、Softmax）、数据变形（Transpose、Reshape）。

# 一、 计算密集型算子的优化思路

核心目标是：让计算单元尽可能忙，减少等待数据的时间，并提高计算效率。

## 利用并行计算

- **多级并行**：充分利用现代处理器的多级并行结构。例如，在GPU上使用成千上万个线程，在CPU上使用多个核心（OpenMP、MPI），以及利用SIMD（单指令多数据流，如CPU的AVX、GPU的warp）指令。

- **切分（Tiling/Blocking）**：将大的计算问题（如大矩阵乘）切分成许多小块，让每个小块能够完全放入速度最快的缓存（L1/L2）中。这样，计算单元在处理这个小块时，几乎不需要等待从主内存中读取数据，极大地提高了数据复用率。

## 优化数据复用

- **矩阵乘的经典优化**：以矩阵乘法为例，通过重排循环顺序和分块，使得加载到缓存中的数据可以被多次计算使用，降低对内存带宽的需求（尽管这是计算密集型算子，但根本还是为了解决“搬运数据给计算单元”的瓶颈）。典型算法如 GotoBLAS 的层次化分块策略。

## 指令集与流水线优化

- **使用更快的指令**：使用向量化的SIMD指令（如FMA，即 fused multiply-add，融合乘加指令）来同时进行多次乘加运算。在可能的情况下，将多个操作合并成一个（如 `a = b*c + d` 直接使用FMA指令，无需中间变量）。
- **指令级并行**：现代处理器可以在一个时钟周期内发射多条指令。通过软件流水和循环展开（Loop Unrolling），减少循环控制的开销，并为编译器提供更多机会来重新排列指令，填满处理器的流水线，隐藏指令延迟。

## 算法改进

- **算法重写**：寻找计算复杂度更低的算法。例如，使用Winograd算法或FFT（快速傅里叶变换）来加速特定尺寸的卷积运算，将乘法次数大幅降低。

# 二、 访存密集型算子的优化思路

核心目标是：减少数据搬运的总量，并提高每次搬运的效率。

## 算子融合

**核心思想**：这是最有效的方法之一。将多个连续的、依次对数据处理的访存密集型算子合并成一个新的算子。

示例：A = 输入 + 1 (访存: 读输入，写A) -> B = A * 2 (访存: 读A，写B) -> C = B / 3 (访存: 读B，写C)。如果这三个算子分开执行，需要读写数据三次。如果融合成一个算子 C = (输入 + 1) * 2 / 3，则只需要一次读输入和一次写C，访存量减少为原来的1/3。虽然计算量不变，但整体耗时因访存瓶颈被缓解而大幅下降。

## 提高访存效率

合并内存访问：特别是在GPU上，要确保相邻的线程访问连续的内存地址，以满足“合并访问”的条件。这能让一次内存事务尽可能满载地传输数据。避免非对齐、随机或步长过大的内存访问模式。

利用局部性：

- 时间局部性：如果一个数据被访问，那么它在不久的将来很可能再次被访问。对于像Sum这样的规约操作，尽量在寄存器或局部内存中完成部分求和，再写回全局内存。
- 空间局部性：如果一个数据被访问，它周围的数据也很可能被访问。尽可能按顺序访问内存地址，让缓存预取机制生效。

## 数据结构优化

数据布局：改变数据在内存中的排列方式，使其更适合算子的访问模式。例如，将结构体数组（AoS）改为数组结构体（SoA）。当需要访问所有数据的某个特定字段时（如坐标x），在SoA布局下，这些x值是连续存储的，可以一次加载一大块，缓存利用率更高。

压缩与位宽：在精度允许的情况下，使用更小的数据类型（如float16/int8代替float32），减少单次数据搬运的字节数，相当于变相提高了带宽。

## 减少内存分配与拷贝

- 内存池：避免在循环或频繁调用的算子里动态分配和释放内存。使用内存池来复用内存块。
- 零拷贝/原地操作：如果一个算子操作后，输入数据不再被需要，可以考虑原地操作，直接修改输入数据的内存，而不是分配新内存并写入。例如 `x = x + 1` 而不是 `y = x + 1`。

# 总结

| 优化维度 | 计算密集型算子 | 访存密集型算子 |
| :--- | :--- | :--- |
| 核心瓶颈 | 算力（计算单元的处理速度） | 带宽（内存/缓存的数据传输速度） |
| 主要战场 | 寄存器、L1/L2缓存、计算单元 | DRAM、缓存行、内存总线 |
| 王道策略 | 分块计算，让数据待在最快的存储中 | 算子融合，减少数据来回搬运的次数 |
| 微观技巧 | 循环展开、向量化（SIMD/FMA）、指令重排 | 合并访问、SoA数据布局、利用局部性 |
| 理想目标 | 达到理论算力峰值 | 达到理论带宽峰值 |
